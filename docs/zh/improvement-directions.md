# 改进方向

本仓库覆盖了 12 个递进式 harness 机制，从最小 agent loop 到 worktree 隔离的自治执行。以下方向是现有教学内容的自然延伸，部分已在 README 范围说明中标注为有意省略的生产机制。

## 优先级分组

### P0 — 核心缺失机制

这些是生产 agent harness 不可或缺的组件，值得作为独立课程（s13+）加入。

#### Hook / 事件总线

当前 s12 仅提供最小 append-only 生命周期事件流。生产 harness 需要完整的 Hook 系统：

| Hook | 触发时机 | 用途 |
|------|----------|------|
| `PreToolUse` | 工具执行前 | 权限检查、参数校验、审计日志 |
| `PostToolUse` | 工具执行后 | 结果转换、缓存、指标采集 |
| `SessionStart` | 会话初始化 | 环境准备、身份加载 |
| `SessionEnd` | 会话结束 | 清理、持久化、计费 |
| `ConfigChange` | 配置热更新 | 动态调整行为 |

设计目标：类型安全、可组合（多个 hook 链式执行）、异步友好。

#### 权限治理与信任流程

Agent 的权力越大，权限系统就越关键：

- **声明式权限**：工具级别的 allow/deny 规则，支持 glob 模式匹配文件路径
- **动态审批**：破坏性操作（rm -rf、git push --force）实时提请人工确认
- **信任边界**：跨 worktree、跨 agent、跨主机的安全隔离模型
- **审计轨迹**：完整的操作日志，支持回放和追溯

#### 会话生命周期

当前所有会话是"用完即走"模式。扩展方向：

- **resume**：断线后恢复会话状态（messages、tasks、worktrees）
- **fork**：从历史 checkpoint 分叉出新会话，用于 A/B 实验或回滚
- **快照**：周期性保存会话快照，支持时间点回溯

### P1 — 深度增强

这些方向在现有课程基础上深化，不改变核心架构。

#### MCP 运行时完整实现

s05 的 skill 加载机制是 MCP 的教学简化版。完整生产实现需要：

- **传输层**：stdio、SSE、WebSocket 多种 transport
- **服务发现**：OAuth 2.0 设备授权流、服务注册与健康检查
- **资源订阅**：文件变更 watch、数据库 CDC、GitHub webhook
- **工具市场**：第三方工具注册、版本管理、沙箱执行

#### 上下文管理 2.0

s06 的三层压缩策略可以进一步演进：

- **智能压缩**：基于 token 预算自动选择压缩策略，而非固定层级
- **语义摘要**：用模型对历史做结构化摘要（决策树、关键事实、待办事项）
- **分级检索**：热（当前窗口）/ 温（近期摘要）/ 冷（长期归档）三级存储
- **缓存感知**：利用 prompt caching 的 TTL 特性优化压缩时机

#### 工具生态扩展

核心 agent loop 不变，新工具只需注册 dispatch handler：

| 工具 | 用途 | 教学价值 |
|------|------|----------|
| 浏览器控制 | 网页抓取、UI 测试、可视化验证 | DOM 交互、iframe 通信 |
| 数据库查询 | 结构化数据存取 | 连接池、事务、ORM |
| 向量检索 | RAG、相似度搜索 | embedding、HNSW、rerank |
| 图像理解 | 截屏分析、图表解读 | 多模态 tool result |
| 网络请求 | API 调用、Webhook 触发 | HTTP 客户端、重试、限流 |

### P2 — 生态与集成

这些方向将 harness 从 CLI 工具扩展到更广阔的应用场景。

#### 团队协议升级

s09-s11 的 JSONL 邮箱协议是教学实现。生产级改进：

- **消息可靠性**：ACK 重试、死信队列、幂等处理
- **协议版本**：兼容多版本 agent 共存
- **发现机制**：agent 注册、能力广播、动态路由
- **联邦团队**：跨机器、跨网络的 agent 协作

#### 外部 IM 通道

参考 [claw0](https://github.com/shareAI-lab/claw0) 的模式，接入多渠道：

- Slack / Discord / Telegram / WhatsApp / Email
- 双向通信：agent 主动推送通知，用户通过 IM 下达指令
- 富媒体支持：图片预览、文件上传、消息 thread

#### 持续记忆

当前记忆仅限于 session 内。跨会话记忆需要：

- **事实存储**：用户偏好、项目约定、决策记录
- **经验积累**：过去成功/失败的 action 序列，用于 Few-shot 提示
- **遗忘机制**：信息老化、冲突消解、隐私删除

### P3 — 工程化与质量

#### Agent 行为测试框架

Agent 的不确定性使传统单元测试不够用。需要专门框架：

- **录制回放**：记录真实 interaction，回放时 mock LLM 输出
- **断言工具**：验证工具调用序列、参数范围、副作用
- **对抗测试**：模糊输入、断网、超时、API 报错等异常场景
- **基线比较**：回归测试时对比行为差异

#### 观测性

- **Token 计量**：按 session、task、tool 维度追踪消耗
- **性能剖析**：工具执行耗时、LLM 响应时间、上下文窗口利用率
- **失败模式分析**：高频错误、循环检测、退化行为预警

#### 可重复构建

- 确定性依赖锁定（lockfile、container image）
- 环境即代码（devcontainer、Nix flake）
- 端到端 CI（类型检查 → 单元测试 → 集成测试 → 行为测试）

## 学习路径建议

```
s01-s12 完成后
│
├── P0: Hook 总线 → 权限治理 → 会话生命周期
│     (核心缺失，顺序依赖)
│
├── P1: MCP 完整实现 → 上下文 2.0 → 工具扩展
│     (深度增强，可并行)
│
├── P2: 团队协议升级 → IM 通道 → 持续记忆
│     (生态扩展，需要 P0 的部分基础)
│
└── P3: 测试框架 → 观测性 → 可重复构建
      (工程化，可随时开始)
```

每个方向保持本仓库的核心教学原则：**心智模型优先，一个文件一个概念，可独立运行的最小实现。**
